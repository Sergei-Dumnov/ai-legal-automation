The technology stack is the centrepiece

Each component has a specific job in the chain from guided input to final legal document. The point is not to sound technical for sport. The point is that legal output quality depends on architecture.

R Shiny Application framework

Provides a guided enterprise-grade user experience so the AI receives precise structured instructions. No open prompt box, no vague user behaviour, no “please be a lawyer” nonsense. Just controlled intake.

Claude API (Claude Sonnet 4.6) Intelligence engine

Used for complex structured generation with a large context window. Strong enough for long-form legal composition while still following a schema rather than wandering off into decorative ambiguity.

Claude Skills / SKILL.md Legal intelligence layer

Loads explicit Australian Commonwealth drafting conventions into every generation call: party block logic, recitals, numbering, sign blocks, defined terms, capitalisation, and other details the model should not be left to guess.

python-docx + LibreOffice Rendering layer

Transforms structured output into a professionally formatted Word document with A4 layout, Times New Roman, justified body text, hanging indents, correct blocks, and headless PDF conversion.

JSON block schema Document contract

Defined block types — title, preamble, recital, clause, subclause, separator, party block, and sign block — give the renderer precise control. The AI emits structure; the application emits the document.

From intake to signed-looking document

The workflow is intentionally structured so the application behaves like a drafting system rather than a conversational toy.

1

Guided intake in the app

The user answers targeted questions about parties, purpose, term, jurisdiction, and commercial details. That structured intake is what makes the downstream output consistent.

2

Legal skill injection

Australian Commonwealth drafting guidance from SKILL.md is included in the generation context for every run, so legal conventions are provided explicitly rather than left to inference.

3

Structured JSON generation

The model returns block-level structure across defined block types, which prevents the renderer from guessing how the document should be assembled.

4

Renderer creates DOCX + PDF

python-docx applies legal formatting rules and LibreOffice handles headless PDF conversion. Not glamorous, but it is where professional output actually happens.
